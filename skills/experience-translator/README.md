# experience-translator

> 经历不是没价值。  
> 只是缺少岗位语言。

**Translate one real experience into role language.**

很多人不是没经历，而是不知道：

- 岗位到底在听什么
- 自己这段经历哪里有用
- 怎么讲才像真实面试，而不是 AI 套话

`experience-translator` 做的事情很窄：  
给它一个 **JD** 和一段 **真实经历**，它会帮你把这段经历翻译成岗位能理解的表达。

## What it does

Paste:

- a target JD
- one real experience

Get:

- transferable evidence
- better positioning
- natural spoken answer
- STAR version if needed
- follow-up answer angles
- saved experience notes for future JD reuse

## Before / After

**Before**

```text
我做了一个校园服务平台项目。
```

**After**

```text
这段经历不只是“做了一个校园项目”。

更适合往产品运营方向讲的是：
我当时需要把很多零散需求先排出优先级，再决定哪些功能先跑起来。
最难的是需求一直在变，所以我一边和同学确认真实使用场景，
一边把流程先落地，再根据反馈调整。
```

这才是它想解决的问题：  
不是把经历包装得更大，而是把真实细节讲得更像岗位需要的能力证据。

## Human Interview Style

`experience-translator` prefers human interview language over polished corporate wording.

It tries to:

- use spoken language
- include real difficulty
- show decision-making
- explain what changed during the process
- avoid abstract labels like "strong communication skills"

Less like this:

```text
我具备较强的沟通协调能力，负责推进项目落地。
```

More like this:

```text
当时最卡的是大家对需求优先级不一致。
我先把必须上线和可以后置的部分拆开，
再和成员确认每一块谁来推进。
```

## Why this exists

普通经历也可以有岗位价值。  
关键是别硬讲，别套话，别假装自己做过没做过的事。

把真实经历翻译成岗位语言，才是面试准备里最值得做的一步。
